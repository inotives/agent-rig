# AgentRig 0.1.3 Release Checklist

## Scope

Release `0.1.3` as a patch release for planner workflow hardening, resume context, and findings notes.

## Pre-Release

- [ ] Confirm working tree is clean.
- [ ] Confirm package version is `0.1.3`.
- [ ] Confirm `CHANGELOG.md` has a `0.1.3` entry.
- [ ] Confirm `docs/release/0.1.3-notes.md` is ready.

## Validation

```bash
npm test
npm --cache /tmp/agent-rig-npm-cache pack --dry-run
git diff --check
```

Local tarball smoke test:

```bash
npm pack --pack-destination /tmp
tmp="$(mktemp -d /tmp/agent-rig-0.1.3.XXXXXX)"
cd "$tmp"
npm --cache /tmp/agent-rig-npm-cache install /tmp/inotives-agent-rig-0.1.3.tgz
AGENT_RIG_SKIP_SKILLS=1 npx agent-rig init --yes
npx agent-rig --version
npx agent-rig validate
```

Resume-context smoke test:

```bash
repo="$(pwd)"
tmp="$(mktemp -d /tmp/agent-rig-start-0.1.3.XXXXXX)"
cd "$tmp"
AGENT_RIG_SKIP_SKILLS=1 node "$repo/dist/index.js" init --yes
AGENT_RIG_SKIP_SKILLS=1 node "$repo/dist/index.js" add planner --role planner
mkdir -p .agent-rig/_shared/handoff_logs .agent-rig/_shared/notes
touch .agent-rig/_shared/handoff_logs/2026-07-06-1010_s1_claude_planner.md
touch .agent-rig/_shared/notes/2026-07-06-0945_worker_later-note.md
node "$repo/dist/index.js" start --agent worker
```

## Tag

- [ ] Commit release preparation.
- [ ] Create tag `v0.1.3`.
- [ ] Push release commit to `origin/main`.
- [ ] Push tag `v0.1.3`.

```bash
git tag v0.1.3
git push origin main
git push origin v0.1.3
```

## Publish

```bash
npm publish --access public
```

Publishing may require npm browser authentication for two-factor auth.

## Post-Publish Verification

```bash
npm view @inotives/agent-rig version
npx -y @inotives/agent-rig@0.1.3 --version
tmp="$(mktemp -d /tmp/agent-rig-npm-0.1.3.XXXXXX)"
cd "$tmp"
AGENT_RIG_SKIP_SKILLS=1 npx -y @inotives/agent-rig@0.1.3 init --yes
npx -y @inotives/agent-rig@0.1.3 validate
```

- [ ] npm registry reports `0.1.3`.
- [ ] npx reports `0.1.3`.
- [ ] npx scaffold and validate succeed.

## GitHub Release

After npm publish succeeds:

```bash
gh release create v0.1.3 --title "v0.1.3" --notes-file docs/release/0.1.3-notes.md
```
