# AgentRig 0.1.2 Release Checklist

## Scope

Release `0.1.2` as a patch release for shared task lifecycle improvements and optional GitHub Issues backlog sync.

## Pre-Release

- [ ] Confirm working tree is clean.
- [ ] Confirm package version is `0.1.2`.
- [ ] Confirm `CHANGELOG.md` has a `0.1.2` entry.
- [ ] Confirm `docs/release/0.1.2-notes.md` is ready.

## Validation

```bash
npm test
npm --cache /tmp/agent-rig-npm-cache pack --dry-run
git diff --check
```

Local tarball smoke test:

```bash
npm pack --pack-destination /tmp
tmp="$(mktemp -d /tmp/agent-rig-0.1.2.XXXXXX)"
cd "$tmp"
npm --cache /tmp/agent-rig-npm-cache install /tmp/inotives-agent-rig-0.1.2.tgz
AGENT_RIG_SKIP_SKILLS=1 npx agent-rig init --yes
npx agent-rig --version
npx agent-rig validate
```

GitHub sync smoke test:

```bash
repo="$(pwd)"
tmp="$(mktemp -d /tmp/agent-rig-gh-0.1.2.XXXXXX)"
cd "$tmp"
git init
git remote add origin git@github.com:inotives/agent-rig.git
AGENT_RIG_SKIP_SKILLS=1 node "$repo/dist/index.js" init --yes
node "$repo/dist/index.js" tasks sync github --limit 5 --dry-run --json
```

## Tag

- [ ] Commit release preparation.
- [ ] Create tag `v0.1.2`.
- [ ] Push release commit to `origin/main`.
- [ ] Push tag `v0.1.2`.

```bash
git tag v0.1.2
git push origin main
git push origin v0.1.2
```

## Publish

```bash
npm publish --access public
```

Publishing may require npm browser authentication for two-factor auth.

## Post-Publish Verification

```bash
npm view @inotives/agent-rig version
npx -y @inotives/agent-rig@0.1.2 --version
tmp="$(mktemp -d /tmp/agent-rig-npm-0.1.2.XXXXXX)"
cd "$tmp"
AGENT_RIG_SKIP_SKILLS=1 npx -y @inotives/agent-rig@0.1.2 init --yes
npx -y @inotives/agent-rig@0.1.2 validate
```

- [ ] npm registry reports `0.1.2`.
- [ ] npx reports `0.1.2`.
- [ ] npx scaffold and validate succeed.

## GitHub Release

After npm publish succeeds:

```bash
gh release create v0.1.2 --title "v0.1.2" --notes-file docs/release/0.1.2-notes.md
```
