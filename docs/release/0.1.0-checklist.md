# AgentRig 0.1.0 Release Checklist

## Account

- [ ] Create npm account.
- [ ] Verify npm account email.
- [ ] Enable npm two-factor authentication.
- [ ] Run `npm login`.
- [ ] Run `npm whoami` and confirm the intended account.

## Package Name

- [ ] Run `npm view agent-rig name version --json`.
- [ ] Confirm the expected result is `E404 Not Found`.
- [ ] If `agent-rig` exists, stop and decide on a scoped package fallback.

## Repository

- [ ] Confirm `git status -sb` is clean.
- [ ] Confirm `package.json` version is `0.1.0`.
- [ ] Confirm release metadata is present in `package.json`.
- [ ] Confirm README install and MVP usage instructions are current.

## Validation

- [ ] Run `npm test`.
- [ ] Run `npm --cache /tmp/agent-rig-npm-cache pack --dry-run`.
- [ ] Run `npm pack`.
- [ ] Run project-local tarball install smoke.
- [ ] Run global tarball install smoke.

## Publish

- [ ] Create tag `v0.1.0`.
- [ ] Push tag `v0.1.0`.
- [ ] Run `npm publish --access public`.
- [ ] Run `npm view agent-rig name version --json`.
- [ ] Install from npm and run `agent-rig --help`.

## GitHub Release

- [ ] Create GitHub release `v0.1.0`.
- [ ] Include npm install command.
- [ ] Include supported MVP commands.
- [ ] Note that AgentRig does not launch Claude, Codex, or OpenCode directly yet.
- [ ] Link to `docs/project_specs.md`.

## Post-Release

- [ ] Configure npm trusted publishing for GitHub Actions.
- [ ] Add tag-based publish workflow.
- [ ] Document the new automated release path.
